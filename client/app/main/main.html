Code Camp 2015

Leaving work at Ultimate Software on a Friday evening, one of my coworkers casually mentioned to me that there was going to be a huge event with 1000+ developers from all over South Florida at a free code camp the next day. Topics were to range from .NET, Azure, JS, Mobile Development, and more, with over 50 speakers giving talks. The best news was that it was literally right down the street from my house, so no excuses not to go!

Arriving at the event this morning, I found myself surrounded by a larger developer community than I suspected South Florida to have. It turned out that nearly 1500 people had registered for the event. Nova Southeastern University's Carl DeSantis Building was teeming with activity, filled with booths, coffee, and bagels as developers milled around the common area, waiting for the sessions to begin.

There were 13 tracks one hour tracks going nearly the entire day. Being an Angular fan, I chose to attend John Papa's "Angular In an Hour" session. I've been a fan of John Papa's Pluralsight videos for nearly three years, when I was first learning Knockout.JS. He has a way of making complex concepts accessible with his casual presentation style. His courses often include custom examples such as his Hot Towel SPA framework which serve to simplify things even further. This Angular session was a great introduction to the popular Google framework that has dominated the MV* JavaScript scene for a couple of years now.

The class was great, but as an advanced Angular user, there was little new material for me to learn there. The next session in that room was to cover JS testing with Jasmine by Jonas Stawski, which is an excellent testing library for front-end code, but I chose to instead check out what was going on over in the Azure track.

Bayer White was giving a talk on "Riding Microsoft's Service Bus to the Cloud and Back". In this class, I got a high level understanding of how to use messaging queues to process transactions, data syncs, message pushes and more. Azure has some nifty features that make this sort of process easy to digest and perhaps as importantly, easy to scale. Following this talk I headed back over to John Papa's second session, which covered Gulp.

Gulp is a build tool based on Node.JS. It can automate any task you give it, from JS linting and minification to TypeScript and CoffeeScript compilation. You can execute templates, to say, generate new controllers or routes on the client side or server side. These templates will scaffold out the code for your new API endpoint. All of these features run in a standard command line or terminal and there are thousands of pre-built ones for you to download and use on NPM. Other advanced uses John showed off were auto-reload, which enables you to edit your JS in your favorite editor and have any open browsers which are running your local webpage get automatically refreshed once you save. These nice features speed up development and make Gulp a powerful tool.

I have been using Grunt for nearly a year now, and John Papa was able to clearly describe the difference between the two tools, illustrating that Gulp runs like a program, with the ability to chain tasks before writing back to the file system. Grunt operates off of config files which are great for getting started, but tend to get unmaintainable as a build system grows in complexity. After seeing the power and simplicity of Gulp, I will definitely be switching to this tool for my workflow.

Other talks I got to check out covered beacons, such as Estimote, and their potential uses in mobile applications. I'm familiar with the uses that cover indoor location, which provides users a simple way to navigate around a building and locate products they would like to find. One of the first uses people seem to jump to, and indeed, that Estimote promotes on their site, is a spam bot for hawking deals to smart phone users as they try to shop in relative peace and anonymity in a retail setting. Hey, sometimes, I love spam. But I found myself pondering potential social applications of location-aware beacons. Could they be used at a club for an on-site Tinder? Can we invent social games that people would play with only people in range of these sensors? I think these sort of social experiences have yet to be fully explored with Internet of Things devices such as this. Technology like this really gets my creative juices flowing!

Another talk covered Xamarin, which I have read about as a great way to make cross platform mobile applications using C# and XAML. For rotaChat, I am heavily considering the use of Xamarin to build out the mobile experience, given that it provides 100% of the native functionality of an Android or iOS app with a language I know and enjoy. This talk reaffirmed to me the capabilities that Xamarin has, but I also recently learned about Facebook's React.js Native library that is soon to be released and open-sourced. React Native is supposed to be able to provide the same features as Xamarin, native runtime, cross-platform mobile app development with an HTML/JS core. Exciting times in mobile development, to be sure.

I caught the tail end of a talk by Ibis Arrastia, a Chapter Leader of Girl Develop It (GDI), a society for mentoring women to learn how to code. She was providing great insight to women and showing off a few of the rising female stars in the South Florida tech community.

The last talk I attended covered Azure Machine Learning, a cloud-based service for piping data streams into battle-tested and proven machine learning algorithms, providing predictive analytics for a wide range of business applications. When it comes to Machine Learning, I am like Jon Snow, I know nothing. I was super impressed with Azure's simple, drag-and-drop interface for connecting algorithms and datasets. Within a few minutes, you can produce a public endpoint with generated sample code for integration with your website in three different languages! This talk opened my eyes to the fact that you can know nothing and still make sense out of big data.

Super encouraged by the activity going on in the South Florida .NET Community!
